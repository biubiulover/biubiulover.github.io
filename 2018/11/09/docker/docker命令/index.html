docker命令大全

发表于 2018-11-09 | 分类于 docker

镜像 命令

镜像仓库

search

Search the Docker Hub for images
–从Docker Hub中搜索镜像

pull

Pull an image or a repository from a registry
–从docker hub中下载镜像

push

Push an image or a repository to a registry
–将本地的镜像上传到docker hub中

本地镜像管理

images

List images
–列出本机的所有镜像

tag

Tag an image into a repository
–对镜像进行重命名

rmi

Remove one or more images
–删除镜像

build

Build an image from a Dockerfile
–通过Dockerfile创建镜像

history

Show the history of an image
–显示镜像制作的过程，相当于dockfile

commit

Create a new image from a container’s changes
–通过容器创建本地镜像
注意：如果是要push到docker hub中，注意生成镜像的命名

export

Export a container’s filesystem as a tar archive
–将容器的文件系统打包成tar文件

import

Import the contents from a tarball to create a filesystem image
–根据tar文件的内容新建一个镜像，与之前的export命令相对应

load

Load an image from a tar archive or STDIN
–与下面的save命令相对应，将下面sava命令打包的镜像通过load命令导入

save

Save an image(s) to a tar archive
–将镜像打包，与上面的load命令相对应

容器 命令

容器生命周期管理

run

Run a command in a new container
–让创建的容器立刻进入运行状态，该命令等同于docker create创建容器后再使用docker start启动容器

create

Create a new container
–创建一个新的容器，注意，此时，容器的status只是Created

start

Start one or more stopped containers
–启动容器

stop

Stop a running container
–停止一个运行的容器

kill

Kill a running container
–强制终止容器

关于stop和kill的区别：

docker stop命令给容器中的进程发送SIGTERM信号，默认行为是会导致容器退出，当然，
容器内程序可以捕获该信号并自行处理，例如可以选择忽略。

docker kill则是给容器的进程发送SIGKILL信号，该信号将会使容器必然退出。

restart

Restart a running container
–重启容器

pause

Pause all processes within a container
–暂停容器内的所有进程

此时，通过docker stats可以观察到此时的资源使用情况是固定不变的，
通过docker logs -f也观察不到日志的进一步输出。

unpause

Unpause all processes within a container
–恢复容器内暂停的进程，与pause参数相对应

rm

Remove one or more containers
–删除容器

注意，不可以删除一个运行中的容器，必须先用docker stop或docker kill使其停止。
当然可以强制删除，必须加-f参数
如果要一次性删除所有容器，可使用 docker rm -f docker ps -a -q，其中，-q指的是只列出容器的ID

exec

Run a command in a running container
–用于容器启动之后，执行其它的任务

容器操作

ps

List containers
–列出所有容器

其中docker ps用于查看正在运行的容器，ps -a则用于查看所有容器。

docker ps -l 查询最后一次创建的容器

inspect

Return low-level information on a container or image
–用于查看容器的配置信息，包含容器名、环境变量、运行命令、主机配置、网络配置和数据卷配置等。

attach

Attach to a running container
–将终端依附到容器上

top

Display the running processes of a container
–查看容器中正在运行的进程

logs

Fetch the logs of a container
–用于查看容器的日志，它将输出到标准输出的数据作为日志输出到docker logs命令的终端上。常用于后台型容器。

port

List port mappings or a specific mapping for the CONTAINER
–输出容器端口与宿主机端口的映射情况

wait

Block until a container stops, then print its exit code
–捕捉容器停止时的退出码

执行此命令后，该命令会“hang”在当前终端，直到容器停止，此时，会打印出容器的退出码。

events

Get real time events from the server
–实时输出Docker服务器端的事件，包括容器的创建，启动，关闭等。

容器其他

cp

Copy files/folders from a container to a HOSTDIR or to STDOUT
–在宿主机和容器之间相互COPY文件

diff

Inspect changes on a container’s filesystem
–查看容器内发生改变的文件，以我的mysql容器为例

rename

Rename a container
–更改容器的名字

stats

Display a live stream of container(s) resource usage statistics
–动态显示容器的资源消耗情况，包括：CPU、内存、网络I/O

其他 命令

version

Show the Docker version information
–查看docker的版本

info

Display system-wide information
–查看docker的系统信息

login

Register or log in to a Docker registry
–登录到自己的Docker register，需有Docker Hub的注册账号

logout

Log out from a Docker registry
–退出登录
